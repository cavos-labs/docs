## Overview

When users register with email/password in your app, Cavos sends them a verification email. You can customize these emails from the dashboard to match your brand.

## What You Can Customize

### 1. App Information

**App Name:**
- Used in the email subject and content
- Default: Your app name from dashboard
- Placeholder: `{{app_name}}`

**App Logo:**
- Displayed at the top of the email
- Upload an image or provide a URL
- Recommended size: 64x64px
- Placeholder: `{{app_logo}}`

### 2. Email Configuration

**Sender Name:**
- The name users see as the sender
- Example: "My Awesome App"
- Default: Uses your app name

**Reply-To Address:**
- Where user replies will be sent
- Example: `support@yourdomain.com`
- Optional - if not set, replies go nowhere

<Note>
All emails are sent from `noreply@cavos.xyz` (verified Resend domain). You can customize the **sender name** and **reply-to address**, but not the sender email address.
</Note>

### 3. Email Template

Customize the full HTML template with these placeholders:

| Placeholder | Description | Example |
|-------------|-------------|---------|
| `{{verification_url}}` | The verification link | `https://cavos.xyz/verify?token=...` |
| `{{app_name}}` | Your app name | "My Awesome App" |
| `{{user_email}}` | User's email address | "user@example.com" |
| `{{app_logo}}` | Your app logo URL | "https://..." |

## Customizing from Dashboard

1. Go to **Dashboard ‚Üí Apps ‚Üí [Your App] ‚Üí Email Verification**
2. Click **"Configure"**
3. Edit the fields:
   - **App Name**: Shown in email
   - **App Logo**: Upload or paste URL
   - **Sender Name**: Display name for sender
   - **Reply-To Email**: Where replies go
   - **HTML Template**: Full customization
4. Click **"Show Preview"** to see how it looks
5. Click **"Save Settings"**

## Email Header Format

When users receive your email, they'll see:

```
From: Your App Name <noreply@cavos.xyz>
Reply-To: support@yourdomain.com
To: user@example.com
Subject: Verify your email for Your App Name
```

When they click "Reply", their email client will address it to `support@yourdomain.com`, not `noreply@cavos.xyz`.

## Why Can't I Use My Own Sender Email?

Using a custom sender address (e.g., `noreply@yourdomain.com`) requires:
- Domain verification in Resend
- DNS configuration (SPF, DKIM, DMARC records)
- Per-app Resend API keys
- Complex setup that most developers don't need

The reply-to approach provides:
- ‚úÖ Zero configuration
- ‚úÖ Immediate functionality
- ‚úÖ Maximum deliverability (verified Resend domain)
- ‚úÖ Professional appearance with custom name
- ‚úÖ User replies still reach you

## Testing Your Email Template

1. Register a test account in your app
2. Check the verification email you receive
3. Test all placeholders are replaced correctly
4. Test the verification link works
5. Test responsive design on mobile

<Tip>
Use a test email account (like Gmail) to check if your email lands in spam or inbox.
</Tip>

## Rate Limits

To prevent abuse:
- Users can request a verification email every **60 seconds**
- Resend endpoint: **3 requests per 10 minutes** per email

## Best Practices

### Subject Line
Keep it short and clear:
- ‚úÖ "Verify your email for MyApp"
- ‚ùå "URGENT ACTION REQUIRED!!!"

### Content
- Be concise and friendly
- Make the verification button prominent
- Include the link as plain text (accessibility)
- State expiration time (24 hours)
- Reassure users who didn't sign up

### Design
- Use responsive design (mobile-friendly)
- Test in multiple email clients
- Keep file size small (<100KB)
- Use web-safe fonts
- Provide a text-only version

### Branding
- Match your app's colors
- Use your logo
- Keep your brand voice
- Include support contact info

## API Usage

You can also update email settings programmatically:

```bash
curl -X PATCH https://cavos.xyz/api/apps/{app_id} \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My App",
    "logo_url": "https://myapp.com/logo.png",
    "email_from_name": "My App Team",
    "email_reply_to": "support@myapp.com",
    "email_template_html": "<!DOCTYPE html>..."
  }'
```

## Troubleshooting

### Emails Not Arriving

1. Check spam folder
2. Verify your reply-to email is valid
3. Check dashboard for any errors
4. Test with different email providers

### Template Not Rendering

1. Validate your HTML
2. Check all placeholders are spelled correctly
3. Test inline CSS (external CSS may not work)
4. Preview in dashboard before saving

### Users Not Clicking Link

1. Make the button more prominent
2. Explain why verification is needed
3. State expiration time clearly
4. Provide a fallback plain text link
